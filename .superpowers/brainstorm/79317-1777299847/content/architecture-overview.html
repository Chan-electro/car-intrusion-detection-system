MQTT TOPICS MAP:
  car/speed      → ESP32 publishes speed (PWM duty %)  → IDS reads
  car/gps        → ESP32 publishes lat/lon every 5s    → IDS reads
  car/ir         → ESP32 publishes IR state            → Dashboard
  hacker/speed   → Hacker injects fake speed           → IDS detects
  hacker/gps     → Hacker injects wrong coordinates    → IDS detects
  ids/alert      → IDS publishes intrusion alerts      → Dashboard
  car/command    → IDS sends speed correction          → ESP32 acts
Scenario 1

🚦 Traffic Signal

IR LED → IR receiver → hardware interrupt stops motor. Hacker cannot override — no software path.

Scenario 2

⚡ Speed IDS

Hacker publishes speed spike → Isolation Forest detects anomaly → IDS corrects via car/command.

Scenario 3

📍 GPS IDS

Hacker publishes fake GPS → IDS compares vs golden route every 5s → deviation alert + lock.
