⚡ ML Pipeline (Speed Anomaly Detection)

PHASE 1 — DATA COLLECTION (Day 1, ~30 min)
──────────────────────────────────────────
1. Run car in normal operation for 10-15 min
2. MQTT subscriber logs car/speed to CSV:
   timestamp, pwm_value, delta_pwm, accel_rate
3. Save as normal_drive.csv (~9000 rows at 100ms)

PHASE 2 — FEATURE ENGINEERING
──────────────────────────────
Features per window (10 readings = 1 second):
  • mean_pwm       — average speed
  • std_pwm        — smoothness
  • max_delta_pwm  — biggest jump
  • accel_rate     — rate of change
  • window_range   — max-min in window

PHASE 3 — MODEL TRAINING (sklearn)
────────────────────────────────────
from sklearn.ensemble import IsolationForest
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X = scaler.fit_transform(features)
model = IsolationForest(
    n_estimators=100,
    contamination=0.05,  # 5% anomaly budget
    random_state=42

Model choice: Isolation Forest

Unsupervised. No labeled attack data needed. Trains only on normal. Ideal for Day 1 collect → Day 2 train flow.

Backup: Z-Score threshold

If ML feels risky: mean ± 3σ rule. delta_pwm > 80 in 100ms = intrusion. Zero training needed.

False positive guard

Require 3 consecutive anomaly detections before triggering alert. Prevents single noisy reading = false alarm.
